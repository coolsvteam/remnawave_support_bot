# 🎫 Universal Support Bot for RemnaWave

Бот технической поддержки для VPN-панелей на базе RemnaWave. 
Позволяет организовать поддержку пользователей через форумные группы (Topics) в Telegram с автоматическим подтягиванием данных о подписке из базы данных панели.

## ✨ Возможности
* **Интеграция с БД RemnaWave:** При открытии тикета бот показывает статус подписки, баланс и остаток трафика пользователя.
* **Форумная система:** Каждое обращение создает отдельную тему (Topic) в админской группе.
* **Мастер установки:** Удобный скрипт setup.sh для быстрой настройки и запуска без танцев с бубном.
* **Локальная БД (SQLite):** Хранение истории тикетов и локальная система банов.

## 🚀 Установка за 1 минуту

Мы подготовили интерактивный мастер установки, который сам всё проверит и настроит:

`git clone https://github.com/GH-Sa1n/remnawave_bedolaga_support_bot.git`
`cd remnawave_bedolaga_support_bot`
`chmod +x setup.sh`
`./setup.sh`

### Шпаргалка: что спросит установщик
1. **Токен бота:** Получите его у @BotFather.
2. **ID админ-группы:** Всегда начинается с -100 (например, -100123456789). Можно узнать, добавив в группу @RawDataBot.
   > **Важно:** В настройках группы обязательно включите тумблер **«Темы» (Topics)** и дайте боту права администратора!
3. **ID темы банов:** Создайте в группе тему «Логи» или «Баны». Напишите туда сообщение и скопируйте ссылку на него (https://t.me/c/123.../42). Цифра в самом конце (42) — это ID темы.
4. **Данные БД RemnaWave:** Потребуется пароль от базы и имя Docker-сети панели (узнать сеть можно командой docker network ls).

## 💡 Использование
1. Пользователь пишет боту и нажимает **«🎫 Открыть новый тикет»**.
2. В админской группе создается новая тема с ID тикета и карточкой клиента.
3. Администраторы пишут ответы прямо в эту тему. Бот пересылает их пользователю.
4. Для закрытия тикета или блокировки спамера используйте inline-кнопки под первым сообщением в теме.
